# 原子化笔记

一种将知识拆解为最小可管理单元的笔记方法。

## 核心理念

### 为什么需要原子化？

| 问题 | 传统笔记 | 原子化笔记 |
|------|----------|------------|
| **整合** | 信息散落在不同载体 | 按概念聚合多源信息 |
| **迭代** | 笔记写完即弃 | 持续更新、深化 |
| **反馈** | 难以感知进步 | 每学一个概念就有获得感 |

!!! tip "核心隐喻"
    - 知识如水，载体如杯——目的是喝水，而非收集杯子
    - 手指指向月亮，而非指向月亮的手指

### 三大原则

#### 1. 一条笔记只做一件事

每条笔记只记录一个**意义簇**（原子）：

- 一个概念
- 一个事实
- 一个经验
- 一个想法
- 一个问题

!!! example "示例"
    在一本书里看到「一个现象」或「一种行为」，就以它为标题建立笔记。

**动态拆分**：内容多了再考虑拆分，不必一开始就过度细分。

#### 2. 一条笔记做好一件事

!!! success "好笔记的特征"
    - **说自己的话**：用自己的话语体系表达
    - **有例子**：搭配具体的例子和类比
    - **有链接**：与其他知识点建立联系
    - **常青**：不断迭代、修改、生长

#### 3. 模板化与标准化

采用一致的笔记结构，便于检索和扩展：

```
# 概念名称

## 定义
[用自己的话解释]

## 例子
[具体案例]

## 关联
[相关概念1] | [相关概念2]

## 来源
[参考资料]
```
